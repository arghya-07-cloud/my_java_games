# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report
it privately so we can resolve it safely. **Do not create a public
GitHub Issue**, as this could expose users to potential risks.

### How to Report

Please send an email to:\
**arghya.open7@gmail.com**

Include the following details: - A clear description of the
vulnerability - Steps to reproduce (if possible) - Any potential impact
you suspect - Suggested fix (optional)

You will receive an acknowledgment within **48--72 hours**.

------------------------------------------------------------------------

## Supported Versions

Only the latest version of the project is officially supported for
security updates.

  Version   Supported
  --------- -----------
  Latest    Yes
  Older     No

------------------------------------------------------------------------

## Disclosure Process

1.  You privately report the vulnerability.\
2.  We investigate and validate the issue.\
3.  We develop and test a fix.\
4.  A patch is released.\
5.  A public advisory may be issued after users have had time to update.

------------------------------------------------------------------------

## Safe Harbor

This project follows a *good-faith security research* policy. As long as
you: - Make a good-faith effort to avoid privacy violations\
- Avoid destruction of data\
- Avoid disrupting services\
- Report the vulnerability privately

You will not face legal action.

------------------------------------------------------------------------

Thank you for helping keep this project secure!
